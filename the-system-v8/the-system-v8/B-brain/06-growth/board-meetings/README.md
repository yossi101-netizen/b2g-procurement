---

## How to Read a Board Brief

Every brief has the same structure. Monday morning read time: 5 minutes.

1. **Session result** — top of file. APPROVED or NO SURVIVOR. If no survivor, no action required.
2. **Approved Initiative** — what to execute. CEO next step is the last line of this section.
3. **Developer Ready Package** — if present, forward to developer immediately.
4. **Killed Proposals** — review takes 30 seconds. One-sentence kill reason per proposal.
5. **Dissent on Record** — read this before acting on any approved initiative. The Red Team's objection and the statistical base rate are preserved verbatim. They are not obstacles — they are the calibration mechanism.
6. **System Health** — check the kill rate. If amber or red, flag before next Sunday.
