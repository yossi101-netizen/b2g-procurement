# System Map

How the ABC-TOM system works, in one page.

---

## The Big Picture

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│   YOU paste a PROMPT (T-tools)                      │
│     ↓                                               │
│   ADAM (COO) reads your CORE (C-core)               │
│     ↓                                               │
│   ADAM assigns AGENTS (A-agents)                    │
│     ↓                                               │
│   AGENTS use SKILLS (T-tools) and                   │
│   follow WORKFLOWS (T-tools)                        │
│     ↓                                               │
│   Output lands in OUTPUT (O-output)                 │
│     ↓                                               │
│   Learnings go to MEMORY (M-memory)                 │
│     ↓                                               │
│   Best patterns get promoted back to CORE           │
│   (This is "The Loop" — your system gets smarter)   │
│                                                     │
└─────────────────────────────────────────────────────┘
```

---

## The 6 Folders

```
ABC-TOM/
│
├── A-agents/          WHO does the work
│   Your team. 9 specialists, each with a clear role.
│
├── B-brain/           WHAT they know about you
│   Your samples, knowledge base, and reference material.
│
├── C-core/            WHO you are
│   Your identity: project brief, voice DNA, ideal customer.
│   This is the foundation everything reads first.
│
├── T-tools/           HOW they work
│   Skills (expert playbooks), Prompts (your shortcuts),
│   and Workflows (multi-step pipelines).
│
├── O-output/          WHAT they produce
│   Every project gets its own folder. Final file at the top,
│   process files in _process/ subfolder.
│
└── M-memory/          WHAT they learned
    Patterns, decisions, feedback. Gets smarter over time.
```

## The 3 Tracks

```
CONTENT TRACK                    SALES TRACK
(Create & Publish)               (Persuade & Close)

You → Adam                       You → Adam
  ↓                                ↓
Researcher (finds angles)        Strategist (analyzes client)
  ↓                                ↓
Copywriter (writes draft)        Copywriter (writes proposal)
  ↓                                ↓
Gatekeeper (reviews)             Gatekeeper (reviews)
  ↓                                ↓
Final post/blog/newsletter       Final proposal/email


THINKING TRACK: DECISIONS        THINKING TRACK: SUMMARIES
(Analyze & Decide)               (Extract & Organize)

You → Adam                       You → Adam
  ↓                                ↓
Strategist (analyzes options)    Analyst (extracts key info)
  ↓                                ↓
Devil's Advocate (challenges)    Copywriter (polishes)
  ↓                                ↓
Chief of Staff (synthesizes)     Gatekeeper (reviews)
  ↓                                ↓
Gatekeeper (reviews)             Final summary/notes/PRD
  ↓
Final decision brief
```

## The Loop (How It Gets Smarter)

```
Round 1: You create content → agents learn your preferences
                                        ↓
                              M-memory/learning-log.md
                                        ↓
Round 2: Agents read past learnings → better output
                                        ↓
                              Pattern proven 3+ times?
                                        ↓
                    YES → Promote to C-core/voice-dna.md
                              (permanent knowledge)
                                        ↓
Round 3+: Even better output, because Core is richer
```

**The system doesn't just produce. It learns.**

## Output Organization

```
O-output/
└── your-project/
    ├── final-[type].md      ← YOUR DELIVERABLE (copy-paste ready)
    └── _process/
        ├── research/draft    ← Internal work
        └── review files      ← Quality checks
```

Open any project folder. The final file is right there. Everything else is tucked away.
