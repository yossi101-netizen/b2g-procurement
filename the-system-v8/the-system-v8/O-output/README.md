# O-output: Your Work

Everything your agents create lives here.

---

## Folder Structure

```
O-output/
├── 01-linkedin-post-productivity/
│   ├── copywriter-draft.md
│   ├── gatekeeper-review.md
│   └── final-post.md
├── 02-twitter-thread-habits/
│   ├── draft-v1.md
│   ├── draft-v2.md
│   └── draft-final.md
└── ...
```

---

## Naming Rules

**Folders:** `[number]-[slug]` format
- `01-linkedin-post-productivity`
- `02-email-client-proposal`

**Files:** `[type]-v[number].md` or `[type]-final.md`
- `copywriter-draft.md` → First draft
- `gatekeeper-review.md` → Review notes
- `final-post.md` → Approved version

---

## Workflow

1. Agent creates → saves to `draft-v1.md`
2. Gatekeeper reviews → saves feedback or requests revision
3. Agent revises → saves to `draft-v2.md`
4. Approved → rename to `draft-final.md`
5. Publish

---

## Important

- **Never** save files directly in `O-output/`. Use folders.
- **Version** everything. You might need the earlier draft.
- **Final** means approved. Not just "done."
