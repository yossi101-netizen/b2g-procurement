# Strategic Decision Skill

Turn any decision into a clear, multi-perspective brief you can act on.

## When to Use

- Evaluating business options (build vs buy, expand vs focus, hire vs outsource)
- Preparing for board meetings or leadership discussions
- Making product direction decisions
- Assessing partnerships, investments, or market entries
- Any decision where you need more than gut feeling

---

## Decision Brief Basics

**What makes a good decision brief:**
- Frames the decision in one sentence (not a paragraph)
- Includes multiple perspectives (not just the optimistic one)
- Uses numbers, not adjectives
- Ends with a clear recommendation AND what you still don't know
- Scannable in 2 minutes

**Length sweet spot:** 500-800 words
- Too short: Missing critical analysis
- Too long: Nobody reads past page 1
- Perfect: One page decision brief with supporting analysis

---

## The Decision Brief Structure

Every decision brief follows this 7-section format:

### Section 1: Decision Required (1 sentence)
```
What must be decided, in plain language.
Not "should we explore..." but "Do we launch X by March or wait until Q3?"
```

Frame it as a clear yes/no or A/B/C choice. If you can't frame it clearly, the decision isn't ready.

### Section 2: Context (2-3 sentences)
```
Why now? What triggered this decision?
What happens if we don't decide?
```

Keep it brief. If you need a full page of context, you're probably not ready to decide.

### Section 3: Options (2-4 options)
```
For each option:
- What it is (1-2 sentences)
- Pros (bullet list)
- Cons (bullet list)
- Estimated impact (numbers — revenue, time, resources)
- Confidence level (High / Medium / Low)
```

Always include "Do Nothing" as an option. Making the cost of inaction explicit changes the conversation.

### Section 4: Risk Assessment
```
| Risk | Likelihood | Impact | What We'd Do |
|------|-----------|--------|-------------|
```

Focus on the 3-5 risks that actually matter. Not every theoretical risk.

### Section 5: Recommendation
```
Go with [Option]. Confidence: [High/Medium/Low].
Why: [2-3 sentences integrating both the optimistic and cautious perspectives]
Key condition: [What must be true for this to work]
```

Take a stand. "It depends" is not a recommendation.

### Section 6: What We Don't Know
```
1. [Uncertainty] — How to resolve: [action]
2. [Uncertainty] — How to resolve: [action]
```

Honesty about unknowns builds trust. Pretending you know everything kills it.

### Section 7: Next Steps
```
1. [Action] — [Who] — [By when]
2. [Action] — [Who] — [By when]
```

Every brief ends with action. Otherwise it's just an essay.

## Decision Type Templates

### Build vs Buy
```
Decision: Build [X] in-house or use [existing solution]?
Key factors: Cost over 12 months, time to market, customization needs, ongoing maintenance, team capability
Common trap: Underestimating maintenance cost of building in-house
```

### Market Entry
```
Decision: Enter [market/segment] now or wait?
Key factors: Market size, competition, first-mover advantage, resource cost, learning curve
Common trap: Overweighting first-mover advantage (usually smaller than you think)
```

### Hire vs Outsource
```
Decision: Hire [role] full-time or outsource to [freelancer/agency]?
Key factors: Long-term need, cost comparison (18 months, not monthly), knowledge retention, speed to start
Common trap: Comparing monthly cost without factoring management overhead
```

### Product Direction
```
Decision: Build [feature A] or [feature B] next?
Key factors: Customer demand, revenue impact, technical complexity, strategic alignment
Common trap: Building what's easy instead of what's impactful
```

### Partnership Evaluation
```
Decision: Partner with [company/person] or go alone?
Key factors: What each side brings, risk of dependency, exit strategy, cultural fit, economics
Common trap: Getting excited about the upside without defining the exit terms
```

## Writing Rules

### Rule 1: Frame Before You Analyze
```
Bad: Starting with "Here are the options..."
Good: Starting with "The decision: Do we X or Y? We must decide by [date] because [reason]."
```

### Rule 2: Numbers Over Adjectives
```
Bad: "Significant revenue potential"
Good: "Estimated $50-80K additional revenue in year 1"
```

### Rule 3: Label Your Confidence
```
High confidence: "Revenue data from last 3 quarters shows..."
Medium confidence: "Based on comparable companies, we estimate..."
Low confidence: "Our best guess, pending customer interviews..."
```

### Rule 4: Make the Trade-off Explicit
```
Bad: "Option A is better"
Good: "Option A gets us there 3 months faster but costs 2x more. Option B is cheaper but we risk losing the window."
```

## What to Avoid

1. **"Analysis paralysis"** — more analysis won't make uncertainty disappear
2. **Burying the recommendation** — put it upfront, not at the end
3. **Ignoring the "do nothing" option** — inaction has a cost too
4. **Vague risks** — "things might go wrong" helps nobody
5. **No next steps** — a decision without action is just an opinion
6. **Over-confidence** — saying "definitely" when you mean "probably"

---

## Quality Checklist

Before finalizing:

- [ ] Decision framed in one clear sentence?
- [ ] All options have numbers (even estimates)?
- [ ] "Do nothing" option included?
- [ ] Risks are specific (not generic)?
- [ ] Clear recommendation with confidence level?
- [ ] Unknowns acknowledged honestly?
- [ ] Next steps are specific (who, what, when)?
- [ ] Scannable in 2 minutes?
- [ ] Relevant to this specific business (not generic)?
