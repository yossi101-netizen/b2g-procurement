# Blog Post Skill

Write posts that people start AND finish. Structure is everything.

## When to Use

- Writing blog posts or articles (any platform)
- Converting ideas, talks, or threads into long-form content
- Creating thought leadership or educational content

---

## Blog Post Fundamentals

**What separates good posts from great ones:**
- A headline that creates a specific curiosity gap
- An opening that earns the next paragraph
- Subheadings that work as a standalone outline
- One clear takeaway the reader remembers tomorrow

**Ideal length by type:**
- **Quick insight:** 500-800 words (one idea, well explored)
- **Tutorial/How-to:** 1,000-1,500 words (step-by-step, actionable)
- **Deep dive:** 1,500-2,500 words (thorough analysis, multiple angles)

**General rule:** Every paragraph earns its place. If removing it doesn't hurt, remove it.

---

## The Headline

The headline does 80% of the work. If nobody clicks, nobody reads.

### Headline Formulas That Work

**The "How To" with a twist:**
```
How I [achieved result] Without [expected method]
How to [do thing] in [surprisingly short time]
```

**The Number + Promise:**
```
7 [Things] That [Transformed/Changed/Fixed] My [Area]
The 3-Step Framework for [Desired Outcome]
```

**The Counterintuitive:**
```
Why [Common Practice] Is Holding You Back
Stop [Common Advice]. Do This Instead.
```

**The Story Tease:**
```
What Happened When I [Did Unexpected Thing] for [Time Period]
I [Did Bold Thing]. Here's What I Learned.
```

### Headline Rules
1. Be specific (numbers, timeframes, outcomes)
2. Create a curiosity gap (not clickbait — deliver on the promise)
3. Under 70 characters if possible
4. Test: Would YOU click this?

## Post Structure

### Structure 1: The Narrative Post
```
[Headline]

[Hook: Start with a moment, not a setup]

[The Problem: What most people get wrong]

[The Turning Point: What changed]

[The Framework: 3-5 steps or principles]
  - Step 1 with example
  - Step 2 with example
  - Step 3 with example

[Results: What happened when you applied this]

[Takeaway: One sentence the reader remembers]

[CTA: What to do next]
```

### Structure 2: The Tutorial
```
[Headline]

[Hook: The outcome they want]

[Why This Matters: Quick context]

[Prerequisites: What they need before starting]

[Step 1: [Action]]
  - What to do
  - Example or screenshot
  - Common mistake to avoid

[Step 2: [Action]]
  - Same format

[Step 3: [Action]]
  - Same format

[Results: What they should see now]

[FAQ or Troubleshooting: 2-3 common issues]

[Next Steps: Where to go from here]
```

### Structure 3: The Argument
```
[Headline: Contrarian claim]

[Hook: Why this matters now]

[The Common Belief: What everyone thinks]

[Why It's Wrong: Evidence and reasoning]
  - Point 1 with data
  - Point 2 with example
  - Point 3 with logic

[The Alternative: What actually works]

[Proof: Results from applying the alternative]

[Nuance: When the common belief IS right]

[Conclusion: Updated mental model]
```

## Writing Rules for Blog Posts

### Rule 1: The Opening Paragraph
You have 3 sentences to earn their attention. Start with:
- A story moment ("Last Tuesday, I deleted 200 emails without reading them.")
- A bold claim ("Most blog posts fail in the first paragraph.")
- A question that stings ("When's the last time you finished reading a blog post?")

Never start with: background, definitions, "In today's world...", or throat-clearing.

### Rule 2: Subheadings as Scannable Outline
Most readers scan before they read. Your subheadings should tell the full story on their own.

```
Bad subheadings:
- Introduction
- Background
- Analysis
- Conclusion

Good subheadings:
- Why 90% of Cold Emails Get Deleted
- The 3-Second Test Every Email Must Pass
- The Template That Gets 40% Reply Rates
- What to Do When They Don't Reply
```

### Rule 3: One Idea Per Section
Each section under a subheading = one point. Make it, support it, move on.

### Rule 4: Show, Don't Tell
```
Telling: "This framework is very effective."
Showing: "After switching to this framework, our team shipped 3x more features in Q2."
```

### Rule 5: Formatting for Readability
- Short paragraphs (2-4 sentences max)
- Bullet points for lists
- Bold for key terms (sparingly)
- Code blocks or quotes for examples
- White space between sections

### Rule 6: The Ending
Don't just stop. Give them:
1. A single, memorable takeaway
2. A clear next action
3. A reason to share or save the post

## What to Avoid

1. **Throat-clearing introductions** — "In today's fast-paced world..." Delete.
2. **Weak conclusions** — "In conclusion, we can see that..." Rewrite.
3. **Too many ideas** — One post, one core message.
4. **No examples** — Abstract advice without concrete examples = forgettable.
5. **Buried value** — Front-load the good stuff. Don't make them scroll to find it.
6. **Walls of text** — If a paragraph is longer than 4 lines, break it up.

---

## Quality Checklist

Before publishing:

- [ ] Headline creates genuine curiosity?
- [ ] Opening earns the first scroll?
- [ ] Subheadings tell the story on their own?
- [ ] Every section has one clear point?
- [ ] Specific examples (not just abstract advice)?
- [ ] Formatting helps scanning (bullets, bold, white space)?
- [ ] Ending has a clear takeaway and next step?
- [ ] Read it out loud — does it flow?
- [ ] Sounds like you (not like a textbook)?
- [ ] Would you share this yourself?
