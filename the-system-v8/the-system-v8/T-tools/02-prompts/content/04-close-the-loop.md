# Prompt: Close The Loop

You just created your first output. Now let's make the system smarter by learning from what happened. This is The Loop. It's what makes week 10 better than week 1.

Copy and paste this into your chat:

---

```
I just created my first output. Now I want to close The Loop so the system learns and improves.

Adam, please manage this:

1. Read what we just created:
   - Check the latest folder in O-output/ (the one we just made)
   - Read the researcher brief, copywriter draft, gatekeeper review, and final output

2. Read current memory and core files:
   - M-memory/learning-log.md
   - M-memory/feedback.md
   - M-memory/decisions.md
   - C-core/project-brief.md
   - C-core/voice-dna.md
   - C-core/icp-profile.md

3. Log what we learned to M-memory/learning-log.md:
   - What worked well in this output? (voice, structure, hooks, tone)
   - What felt off or needed revision?
   - Any patterns worth remembering for next time?
   - What did the Gatekeeper flag?

4. Check if anything is ready to promote to C-core:

   **From learning-log to voice-dna:**
   - Any voice pattern that worked well? Add it as a rule.
   - Any mistake to avoid? Add it as a "never do this."

   **From feedback to icp-profile:**
   - Any audience language we should capture?

5. Sort B-brain/04-INBOX/:
   - Move any unsorted items to the right subfolder
   - Delete anything no longer relevant
   - Summarize what you sorted

6. Show me a summary:
   - What we learned from this output
   - What got promoted to C-core (if anything)
   - What stayed in Memory (not ready yet)
   - What was sorted from INBOX
   - One specific thing to try differently next time

This is The Loop. Every time you close it, the system gets smarter.
```

---

## Why The Loop Matters

Most AI systems are static. You set them up and they stay the same forever.

The ABC-TOM Loop is different. Every project teaches you something. The Loop captures those lessons and bakes them into the system permanently.

**Week 1:** Memory is empty. Output is rough.
**Week 4:** Memory has patterns. Output improves.
**Week 8:** Strong patterns get promoted to Core. Output sounds like you.
**Week 12:** The system runs itself. You just steer.

The Loop is the difference between "using AI" and "building an AI team."

---

## When to Use This

- Right after creating any output (like now!)
- At the end of each week (pair with bonus prompt 12 - weekly review)
- When your output starts feeling stale
- When Memory files are getting long and need cleanup
