## מהות הפרויקט

**Israel-Hunter V3** היא מערכת אוטומטית לאיתור וסיווג לידים עבור **KritiKaal.com** — פלטפורמת ייצור וסיטונאות ישראלית.

**מטרת העל:** Discovery & Enrichment בלבד — המערכת מאתרת, מסננת ומדרגת לידים פוטנציאליים בשוק הישראלי, ומספקת רשימות מוכנות לעבודה. המערכת אינה מבצעת יצירת קשר יזומה (Outreach) בשום שלב.

## ארכיטקטורת ABC-TOM

### ABC — התשתית

| רכיב | שם | תפקיד |
|------|----|--------|
| **A** | **A-Agents** | הסוכנים — מודולים אוטומטיים המבצעים סריקה, חילוץ נתונים, וסיווג לידים ממקורות שונים |
| **B** | **B-Brain** | הידע הטכני — מאגר הלוגיקה, כללי הסיווג, מודלי הדירוג, ופרופילי קהל היעד |
| **C** | **C-Core** | ה-DNA העסקי — הגדרות KritiKaal, קריטריוני כשירות ליד, ומדיניות הפרויקט |

### TOM — הביצוע

| רכיב | שם | תפקיד |
|------|----|--------|
| **T** | **T-Tools** | סקריפטים ופרומפטים — הכלים הקונקרטיים שכל Agent משתמש בהם לביצוע משימות |
| **O** | **O-Output** | הלידים המופקים — פלט Excel עם שלושה tabs: Class A / B / C |
| **M** | **M-Memory** | לוגים ותובנות — תיעוד ריצות, שגיאות, ואנליזה לשיפור מתמיד |

## אסטרטגיית Market Exhaustion

במקום מכסה קבועה (50-100/שבוע), המערכת פועלת לפי מודל מיצוי:

```
לכל query set פעיל:
    1. הרץ את כל השאילתות
    2. בדוק כל domain מול ה-DB לפני עיבוד
    3. אם domain ידוע → דלג
    4. אם אחוז הדומיינים החדשים מסך התוצאות < 10% → 
       query set זה "מוצה" — עבור לקבוצת שאילתות חדשה
    5. M-Memory מתעד אילו query sets מוצו
```
