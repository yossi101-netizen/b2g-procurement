# Israel-Hunter V3 — Target Audience & Lead Classification
**גרסה:** 2.0 | **עודכן:** 2026-04-09 | **סיבת עדכון:** Strategic Pivot Phase 1 — Volume Buyer Targeting

---

## מטרת הקובץ

קובץ זה מגדיר את **קריטריוני הסיווג הקשיחים** שלפיהם האלגוריתם מדרג כל ליד שנאסף.
הסיווג הוא אוטומטי: Tier-1 (heuristics מקומיות) → Tier-2 (LLM — הבורר הסופי).

---

## עיקרון-העל: Volume הוא המדד היחיד

KritiKaal מחפשת עסקים שמזיזים **כמויות** — עסקים שמייבאים, מפיצים, מסיטנאים, או מייצרים לפי הזמנה בסדרי גודל. עסק B2C קטן שמוכר מוצר אחד בפעם אינו מטרה, גם אם הוא מייצר בעצמו.

---

## מפת הסיווג

```
ליד נאסף
    │
    ├─► Class A — QUALIFY       (Volume buyer — מטרת העל)
    ├─► Class B — REVIEW        (פוטנציאל לא מאושר — לאימות ידני)
    └─► Class C — DISQUALIFY    (B2C dropshipper — נשמר לניתוח בלבד)
```

---

## Class A — Volume Buyers (מטרת העל)

**עדיפות:** ראשית. לידים אלו מועברים ישירות לרשימת ה-Output הראשית.

### הגדרה
עסק ישראלי שפועל בנפח — מייבא, מפיץ, מסיטנאי, או מייצר כ-OEM. ה-LLM הוא הבורר הסופי לסיווג זה.

### חמש קטגוריות כשירות — כל אחת מהן מספיקה לבדה

| # | קטגוריה | תיאור | סיגנלים מרכזיים |
|---|----------|-------|-----------------|
| **1** | **יבואן** | מייבא מוצרים מחו״ל ומחזיק מלאי בישראל. כולל יבואנים מאיטליה, טורקיה, ספרד, סין — מוצא הסחורה הוא **הזדמנות**, לא פסילה | "יבוא בלעדי", "נציג רשמי", "יבואן מוצרי...", "מיובא מ-..." |
| **2** | **OEM Brand** | מעצב בישראל, מייצר בחו״ל לפי מפרט עצמי. **תמיד Class A** — ללא יוצא מן הכלל | "קולקציה", "עיצוב ישראלי", "מיוצר בטורקיה/סין לפי הזמנה", "מותג עצמאי" |
| **3** | **סיטונאי** | מוכר לעסקים אחרים בכמויות. מחזיק מלאי לאספקה מהירה | "סיטונאות", "מחיר סיטונאי", "מכירה לסיטונאים", "מינימום הזמנה", "מכירה בכמויות" |
| **4** | **מפיץ B2B** | מפיץ מורשה של מותג לשוק הישראלי, מוכר לרשתות / עסקים | "נציג רשמי ובלעדי", "מותג...", "אספקה לרשתות", "לקוחות עסקיים" |
| **5** | **יצרן מקומי** | ייצור פיזי בישראל | "מפעל", "פס ייצור", "מייצרים", "פטנט" |

### Tie-Breaker Rule (כשהסיגנלים מעורבים)

כשאתר נראה כמי שמוכר מוצרים מיובאים אך **חסרה כתובת פיזית ברורה או זמני משלוח**:

```
IF B2B signals present ("סיטונאות", "רכישה מרוכזת", "קולקציה", "אספקה לעסקים")
    → QUALIFIED_A

IF no B2B signals AND looks like small B2C retailer (no volume indicators)
    → QUALIFIED_B_PENDING_VERIFY

IF OEM Brand signals present (own collection + manufactured abroad)
    → QUALIFIED_A  ← תמיד, ללא קשר לשאר הסיגנלים
```

### קוד סטטוס ב-SQLite
```
status = 'QUALIFIED_A'
```

## Class B — Pending Manual Review (משנית)

**עדיפות:** משנית. מסומנים לאימות טלפוני על ידי צוות KritiKaal.

### הגדרה
חנויות איקומרס עצמאיות ללא הוכחת נפח. ייתכן שמחזיקים מלאי חלקי בישראל, אך אין תיעוד מספיק לסיווג A. ה-LLM לא הצליח להבחין בבירור.

### קריטריוני סיווג

| קריטריון | ערך |
|-----------|-----|
| confidence LLM | בין 0.60 ל-0.74 |
| סיגנלי נפח B2B | לא נמצאו |
| דרופשיפ מובהק | לא הוכח |

### קוד סטטוס ב-SQLite
```
status = 'QUALIFIED_B_PENDING_VERIFY'
```

## Class C — Disqualified (פסילה)

**עדיפות:** אפס לאאוטריץ׳, אך **נשמר ב-DB ומיוצא ל-CSV** לבדיקה ידנית של הצוות.

### הגדרה
עסקים שאינם מחזיקים מלאי בישראל ומעבירים הזמנות ישירות לספקים בחו״ל, **או** שה-LLM קיבל confidence נמוך מ-0.60.

### סימני פסילה — אחד מהם מספיק

| סימן | דוגמאות לשון |
|------|--------------|
| **זמן משלוח ארוך** | "14-28 ימי עסקים", "3-5 שבועות" |
| **מחסן בחו״ל** | "משלוח ממחסנינו בסין", "shipped from overseas warehouse" |
| **לוגיסטיקה זרה** | ePacket, AliExpress logistics, USPS, China Post |
| **מונחי דרופשיפינג** | "dropship", "print on demand", "fulfillment center abroad" |
| **confidence נמוך** | LLM confidence < 0.60 |

### הבחנה קריטית — OEM ≠ Dropship
עסק שמייצר לפי מפרט עצמי בחו״ל (OEM) **אינו** דרופשיפר — הוא **תמיד Class A**.
דרופשיפר = מעביר הזמנות ללא שליטה על המוצר. OEM = בעל המוצר, מזמין ייצור.

### קוד סטטוס ב-SQLite
```
status = 'DISQUALIFIED_C'
disqualify_reason = '<הסימן שזוהה>'
```

## סף ה-Confidence של ה-LLM — כלל מחייב לקוד

```
confidence >= 0.75   →  QUALIFIED_A
0.60 <= confidence < 0.75  →  QUALIFIED_B_PENDING_VERIFY
confidence < 0.60    →  DISQUALIFIED_C
```

**חריג יחיד:** OEM Brand מאובחן בוודאות → QUALIFIED_A ללא קשר ל-confidence.

---

## לוגיקת הסיווג האוטומטי (Tier-1 → Tier-2)

```
שלב 1 — Tier-1 (heuristics, ללא עלות):
    • אם טקסט ריק / לא עברי מספיק → UNCLASSIFIED
    • אם יש סיגנל דרופשיפ מובהק ולא מנוגד → DISQUALIFIED_C (conf=0.95)

שלב 2 — Tier-2 (LLM):
    • LLM מקבל טקסט + Tier-1 hints
    • LLM מחזיר status + confidence + signals
    • כלל ה-confidence לעיל מאמת את ה-status
    • OEM override חל אחרי קבלת התשובה
```

## אסטרטגיית גילוי — Market Exhaustion

המערכת פועלת ברצף עד **מיצוי השוק** — אין מכסה שבועית. ריצה מסתיימת כשאין דומיינים חדשים לגלות עבור קבוצת השאילתות הפעילה. הקוד בודק את ה-DB לפני כל INSERT ודולג על דומיינים ידועים.
