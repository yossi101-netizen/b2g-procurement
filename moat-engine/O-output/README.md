# O-output

Ranked opportunity shortlists land here, one file per scoring batch.

Suggested name: `shortlist-YYYYMMDD.md` (or `.csv`). Each row: band, score, name,
the gate/reason, and the axis breakdown. HOT rows are the ones that graduate to a
HuntList item in INDIA-STOCKS (see `../README.md` → Bridge).

Keep the DISCARDs with their reasons — the accumulating "why we passed" log is
how the model gets calibrated and how you avoid re-evaluating the same dead idea
twice.
